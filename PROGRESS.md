## Known Issues and Caveats

- **Non-strategy exits no longer inflate the real-fill gates (2026-07-16, RESOLVED).** `trades` now carries an `origin` discriminator (`strategy` | `adaptive_react` | `rebalance`), and `count_closed_trades` counts STRATEGY fills only. That function gates the DNN real-data trainer AND the RL 500-fill activation (`rl_min_real_fills`, a CLAUDE.md hard rule); both build their features from `bars`, so it is purely a gate, and the question it must answer is "has the POLICY been exercised enough", not "have any fills occurred". An adaptive defensive exit and a sleeve rebalance trim are real fills but not policy decisions. Filtering makes both gates STRICTER (they open later, on fewer but meaningful fills), which is the safe direction for a gate whose job is to say "not yet". The rebalance half of this bug PREDATED the adaptive layer and is fixed by the same change. Caveat: a DB written before the migration has no column to filter on, so `count_closed_trades` falls back to the unfiltered count there rather than crashing; historical rows backfill to `strategy` and any pre-existing rebalance trims stay miscounted, because the information to tell them apart was never recorded.

- Advisory factor *scores* on the default (no-`--bridge`) path are deterministic C++ mocks; the real LLM/dnn/whale scores engage only with `--bridge` + the bridge server up. The learning signal the tuner consumes is real (closed-trade PnL), but the verdicts feeding it out-of-box are stand-ins.
- Shipped `dnn_advisory` champion is still synthetic-trained; `rl_advisory` is untrained (real-fill gate unmet) and shipped off. The real-data trainer now DOES train a challenger once enough history exists (verified against a synthetic-regime run: 47,900 samples, `challenger_recorded`).
